# ViBe - 실시간 강의/회의 플랫폼

🎤 **ViBe**는 교육/협업을 위한 웹 기반 실시간 화상 플랫폼입니다. WebRTC를 기반으로 하여 고품질의 비디오/오디오 통신, 화면 공유, 실시간 채팅, 퀴즈, 질문 정리 기능 등을 지원합니다.

## ✨ 주요 기능

- **🎥 화상/음성 통화**: WebRTC 기반의 고품질 P2P 비디오/오디오 통신
- **🖥️ 화면 공유**: 프레젠테이션이나 화면 내용을 실시간으로 공유
- **💬 실시간 채팅**: 참여자들과 텍스트 기반 실시간 메시지 교환 (이미지 첨부 가능)
- **❓ 퀴즈 기능**: 강의자가 선택지 기반 퀴즈 출제, 실시간 답변 수집
- **✨ 질문 정리 기능**: AI를 활용한 질문 다듬기 (OpenAI API)
- **👥 다중 참여자 지원**: 최대 10명 이상의 동시 참여자 지원
- **🎭 익명화**: 참여자의 이름을 익명으로 변경 가능
- **🎛️ 미디어 제어**: 카메라 및 마이크 ON/OFF 제어
- **📱 반응형 UI**: 데스크톱 및 모바일 환경 지원
- **🔔 실시간 알림**: 채팅 메시지, 참여자 입장/퇴장 알림

## 🛠️ 기술 스택

### 백엔드
- **Node.js** - JavaScript 런타임
- **Express.js** - 웹 서버 프레임워크
- **Socket.IO** - 실시간 양방향 통신

### 프론트엔드
- **HTML5** - 마크업
- **CSS3** - 스타일링
- **JavaScript (Vanilla)** - 클라이언트 로직
- **WebRTC** - P2P 미디어 스트리밍

### 통신 프로토콜
- **WebRTC** - 피어 투 피어 미디어 통신
- **Socket.IO** - 시그널링 (SDP, ICE 교환)

## 📋 사전 요구사항

- **Node.js** 14.0 이상
- **npm** 6.0 이상
- 모던 웹 브라우저 (Chrome, Firefox, Safari, Edge)
- HTTPS 환경 또는 localhost (카메라/마이크 접근 권한 필요)

## 🚀 설치 및 실행

### 1. 저장소 클론
```bash
cd ViBe
```

### 2. 의존성 설치
```bash
npm install
```

### 3. 서버 시작
```bash
# 개발 모드
npm run dev

# 또는 일반 실행
npm start
```

서버는 기본적으로 `http://localhost:3000`에서 실행됩니다.

### 4. 브라우저에서 접속
```
http://localhost:3000
```

## 📖 사용 방법

### 회의 시작하기

1. **로그인 화면** 접속
   - 사용자 이름 입력
   - "새로 생성" 버튼으로 회의 ID 생성 또는 수동 입력
   - "회의 참여" 클릭

2. **기존 회의 참여하기**
   - 사용자 이름 입력
   - 기존 회의 ID 입력
   - "참여" 클릭

### 회의 중 기능 사용

| 기능 | 버튼 | 설명 |
|------|------|------|
| **카메라 제어** | 📹 | 비디오 캡처 ON/OFF |
| **마이크 제어** | 🎤 | 오디오 입력 ON/OFF |
| **화면 공유** | 🖥️ | 화면 공유 시작/종료 |
| **설정** | ⚙️ | 기기 설정 (오디오 입력원 선택 등) |
| **회의 종료** | 📞 | 회의 나가기 |

### 채팅 사용법

1. 우측 사이드바에서 "💬 채팅" 탭 선택
2. 메시지 입력 후 "전송" 버튼 클릭 또는 Enter 키 입력
3. 모든 참여자에게 실시간으로 메시지 전달

### 참여자 확인

- 우측 사이드바의 "👥 참여자" 탭에서 현재 회의의 모든 참여자 확인

## 🔧 주요 기술 설명

### WebRTC (Web Real-Time Communication)
- P2P 기반으로 중개 서버를 거치지 않고 직접 미디어 스트림 전송
- SDP (Session Description Protocol)를 통한 세션 협상
- ICE (Interactive Connectivity Establishment)를 통한 NAT 트래버설

### Socket.IO를 통한 시그널링
- 클라이언트 간 메타데이터 및 시그널링 정보 전달
- 채팅 메시지 실시간 브로드캐스트
- 사용자 입장/퇴장 알림

### 화면 공유
- `getDisplayMedia()` API를 통한 화면 캡처
- 캡처된 스트림을 WebRTC 트랙으로 전송
- 기존 카메라 스트림과 자동 전환

## 🐛 알려진 문제

- 네트워크 지연이 높은 환경에서 음성 지연 발생 가능
- 크로스 브라우저 호환성 테스트가 필요할 수 있음
- STUN 서버만 지원하므로 엄격한 NAT 환경에서 연결 불가능할 수 있음
